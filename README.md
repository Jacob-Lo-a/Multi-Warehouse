## 作業一

### 1. 產品與多倉查詢
1. 提供一支 API 回傳包含產品名稱、料號(SKU)、類別名稱、供應商名稱
2. 必須同步顯示該產品在「所有倉庫」的分布狀況 (例如：台北倉 50 件、桃園倉 20 件)

### 2. 精準庫存異動與事務控制
1. 實作「產品入庫」功能：輸入產品、倉庫與數量
2. 如果該倉庫「已有」該產品：累加數量
3. 如果該倉庫「從未有」該產品：新增一筆庫存紀錄
4. 每次異動都必須自動產生一筆「庫存異動日誌 (StockLogs)」

### 3. 專業 Excel 報表產出
1. 提供「全庫存總表匯出」功能
2. Excel 內容：產品名稱、類別、供應商、各倉儲位置、總庫存量。

### 4. 自動化排程與外部系統整合
1. 每天凌晨 3:00 自動篩選「低庫存 (量 < 10)」的產品
2. 將低庫存清單製成 Excel，並自動透過 SFTP 協議傳送到外部供應商的伺服器
